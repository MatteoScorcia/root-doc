sample(mode: str = 'original_structure', n_samples: Optional[Any] = None, device: Optional[Any] = None) → Any¶

Synthesize data. :param mode: One of:

  • sample_fks: Sample the foreign keys with a hierarchical algorithm from the roots to the leafs.

  • sample_subgraphs: Sample the foreign keys by sampling subgraphs from the ones present in the original data. In this case all tables must be synthesized.

  • original_structure: Use the same structure of the original data for the foreign keys.

Parameters:

  • n_samples –

    Dictionary with the number of samples to be generated for each table to be synthesized. If some are missing, the original tables are loaded. However, if a table should not be synthesized, then all its parents cannot be synthesized. The exact behavior of n_samples depends on the synthesis mode: - mode = ‘sample_fks’: The number of synthesized rows is equal to the values of n_samples. - mode = ‘sample_subgraphs’: The values of n_samples are intended as the minimal amount of samples

    to generate for each table. However, since in this case all tables must be synthesized, if the values of n_samples for certain tables are not provided, the original ones will be used.

    • mode = ‘original_structure’: The values of n_samples are ignored.

    If None, the number of samples of the original data are used.

  • device – Optional, device where to perform the computation. If None, the device will be automatically selected.

Returns:

A dictionary with a pandas DataFrame for each table.

report(max_table_size: int = 0, rng: Optional[int] = None, device: Optional[Any] = None, out_dir: str = 'out/') → None¶

Generate the report data. :param max_table_size: Max size for each original table. If 0, take all data. :param rng: A torch Generator or an integer used as seed to extract the subset of original data. If None, a new

generator with a random seed is used.

Parameters:

  • device – Device where to perform the computation. If None, the device will be automatically selected.

  • out_dir – Directory where the report is saved.

__init__(data: Any, intuite_columns: Optional[dict[str, dict[str, Any]]] = None, hidden_units_node: int = 32, aggregator_node: str = 'add_attention', alpha_node: float = 1.0, beta_node: float = 1.0, d_latent_node: int = 8, alpha_link_node: float = 1.0, d_link_node: int = 10, temp_node: float = 1.0, hidden_units_edge: list[int] = (128,), activations_edge: str = 'ReLU', batch_norm_edge: bool = False, dropout_edge: float = 0.0, **kwargs: object) → Any¶
Parameters:

  • data – Instance of RelationalData class.

  • intuite_columns – Dictionary containing intuite column instances for a desired selection of columns. Keys are table names, values are dictionaries indexed by column names and with column instances as values.

  • hidden_units_node – Number of hidden units in the Fully Connected blocks of nodes.

  • aggregator_node – Name of the aggregator used to aggregate the encoded vectors in nodes. Options: leaf, gaussian, add_attention.

  • alpha_node – Weight for the update of the loss of a node.

  • beta_node – Weight of KL divergence in the loss of a node.

  • d_latent_node – Latent dimension of a node.

  • alpha_link_node – Weight for the update of the linking loss of a node.

  • d_link_node – Latent dimension for the linking function between nodes.

  • temp_node – Depends on col_type: - Temperature used in sampling categories for categorical columns. - Variance of the output Gaussian distribution for integer and float columns.

  • hidden_units_edge – Number of hidden units for each layer in the fully connected linker.

  • activations_edge – Activation function(s); if a string it will be imported from torch.nn.

  • batch_norm_edge – Flag for batch normalization.

  • dropout_edge – Dropout probability.
